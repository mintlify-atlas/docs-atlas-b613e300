## Setup Steps

<Steps>
  <Step title="Clone the Repository">
    Clone the StudyQuest repository to your local machine:
    
    ```bash
    git clone https://github.com/your-org/study_quest.git
    cd study_quest
    ```
  </Step>
  
  <Step title="Install Flutter Dependencies">
    Install all required Flutter packages:
    
    ```bash
    flutter pub get
    ```
    
    This will install all dependencies defined in `pubspec.yaml`:
    
    ```yaml pubspec.yaml
    dependencies:
      flutter_bloc: ^8.1.3      # State management
      supabase_flutter: ^2.8.3  # Backend
      get_it: ^7.6.0            # Dependency injection
      dartz: ^0.10.1            # Functional error handling
      file_picker: ^8.0.0       # PDF file selection
    ```
  </Step>
  
  <Step title="Create Supabase Project">
    1. Go to [supabase.com](https://supabase.com) and create a new project
    2. Note your project URL and anon key from **Settings > API**
    3. Create a storage bucket named `pdfs`:
       - Go to **Storage** in the Supabase dashboard
       - Click **Create bucket**
       - Name it `pdfs` and make it public
    
    <Warning>
      Keep your Supabase keys secure. Never commit them to version control.
    </Warning>
  </Step>
  
  <Step title="Set Up Database Tables">
    Run this SQL in the Supabase SQL editor (**Database > SQL Editor**):
    
    ```sql
    -- Documents table
    CREATE TABLE documents (
      id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
      user_id UUID REFERENCES auth.users NOT NULL,
      title TEXT NOT NULL,
      file_url TEXT NOT NULL,
      summary_text TEXT,
      status TEXT DEFAULT 'processing',
      created_at TIMESTAMP DEFAULT NOW()
    );

    -- Topics table
    CREATE TABLE topics (
      id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
      document_id UUID REFERENCES documents ON DELETE CASCADE,
      title TEXT NOT NULL,
      description TEXT,
      order_index INTEGER DEFAULT 0,
      created_at TIMESTAMP DEFAULT NOW()
    );

    -- Flashcards table
    CREATE TABLE flashcards (
      id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
      document_id UUID REFERENCES documents ON DELETE CASCADE,
      topic_id UUID REFERENCES topics ON DELETE CASCADE,
      front_text TEXT NOT NULL,
      back_text TEXT NOT NULL,
      mastery_level INTEGER DEFAULT 1,
      created_at TIMESTAMP DEFAULT NOW()
    );

    -- Quizzes table
    CREATE TABLE quizzes (
      id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
      document_id UUID REFERENCES documents ON DELETE CASCADE,
      topic_id UUID REFERENCES topics ON DELETE CASCADE,
      question_text TEXT NOT NULL,
      options JSONB NOT NULL,
      correct_answer_index INTEGER NOT NULL,
      explanation TEXT,
      created_at TIMESTAMP DEFAULT NOW()
    );

    -- Enable Row Level Security
    ALTER TABLE documents ENABLE ROW LEVEL SECURITY;
    ALTER TABLE topics ENABLE ROW LEVEL SECURITY;
    ALTER TABLE flashcards ENABLE ROW LEVEL SECURITY;
    ALTER TABLE quizzes ENABLE ROW LEVEL SECURITY;

    -- Policies for documents
    CREATE POLICY "Users can view own documents" ON documents
      FOR SELECT USING (auth.uid() = user_id);
    
    CREATE POLICY "Users can insert own documents" ON documents
      FOR INSERT WITH CHECK (auth.uid() = user_id);
    ```
  </Step>
  
  <Step title="Configure App Secrets">
    Create a file `lib/core/secrets.dart` with your credentials:
    
    ```dart lib/core/secrets.dart
    class Secrets {
      // Supabase Configuration
      static const String supabaseUrl = 'https://your-project.supabase.co';
      static const String supabaseAnonKey = 'your-anon-key-here';
    }
    ```
    
    <Warning>
      Add `lib/core/secrets.dart` to your `.gitignore` to prevent committing secrets:
      
      ```bash .gitignore
      # Secrets
      lib/core/secrets.dart
      ```
    </Warning>
  </Step>
  
  <Step title="Set Up Python Worker">
    The backend worker processes PDFs using AI. Set it up:
    
    ```bash
    cd backend
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    pip install -r requirements.txt
    ```
    
    Create `backend/.env` with your credentials:
    
    ```bash backend/.env
    SUPABASE_URL=https://your-project.supabase.co
    SUPABASE_KEY=your-service-role-key
    GEMINI_API_KEY=your-gemini-api-key
    ```
    
    <Note>
      Use the **service_role** key from Supabase (Settings > API) for the backend worker, not the anon key.
    </Note>
  </Step>
  
  <Step title="Run the App">
    Start the Flutter app on your device or emulator:
    
    <CodeGroup>
    ```bash iOS
    flutter run -d ios
    ```
    
    ```bash Android
    flutter run -d android
    ```
    
    ```bash Emulator
    # Start any available device
    flutter run
    ```
    </CodeGroup>
    
    The app will initialize Supabase and dependency injection on startup:
    
    ```dart lib/main.dart
    Future<void> main() async {
      WidgetsFlutterBinding.ensureInitialized();

      await NotificationService.init();
      
      // 1. Initialize Supabase
      await Supabase.initialize(
        url: Secrets.supabaseUrl,
        anonKey: Secrets.supabaseAnonKey,
      );

      // 2. Initialize Dependency Injection
      await di.init();

      runApp(const MyApp());
    }
    ```
  </Step>
  
  <Step title="Start the AI Worker">
    In a separate terminal, start the Python worker to process PDFs:
    
    ```bash
    cd backend
    source venv/bin/activate
    python worker.py
    ```
    
    You should see:
    
    ```
    🤖 StudyQuest AI Worker (V4 - Auto-Nombramiento) iniciado...
    Esperando documentos en la cola...
    ```
    
    <Tip>
      For production, deploy the worker to a cloud service like Railway, Render, or Heroku so it runs continuously.
    </Tip>
  </Step>
  
  <Step title="Upload Your First PDF">
    Now test the complete flow:
    
    1. **Create an account** or sign in
    2. **Tap the upload button** (usually a '+' or 'Upload' button)
    3. **Select a PDF** from your device (try a study guide or textbook chapter)
    4. **Wait for processing** - the document status will show "processing"
    5. **Check the worker logs** - you'll see the AI analyzing the PDF:
    
    ```
    🚀 Procesando documento: example.pdf (abc-123)
       ⬇️ Descargando PDF completo...
       📖 Leyendo PDF...
       📝 Generando título corto y resumen global...
       ✂️ Documento dividido en 2 bloques lógicos.
       🧠 IA Analizando bloque 1...
       📑 IA encontró un total de 3 temas en todo el PDF. Guardando en DB...
    🎉 ¡Proceso completado con éxito! El mundo está listo para jugarse.
    ```
    
    6. **Study your material** - once processing completes, tap the document to access flashcards and quizzes
  </Step>
</Steps>

## Verify Everything Works

To confirm your setup is correct:

<Steps>
  <Step title="Check Document Upload">
    The document should appear in your documents list with status "processing", then change to "ready" after the worker finishes.
  </Step>
  
  <Step title="Verify AI Processing">
    In the worker logs, you should see:
    - PDF download confirmation
    - AI analysis of text chunks
    - Topics, flashcards, and quizzes being saved
    - Success message when complete
  </Step>
  
  <Step title="Test Flashcards">
    Open the processed document and:
    - View generated flashcards
    - Flip cards to see answers
    - Mark cards as mastered
  </Step>
  
  <Step title="Take a Quiz">
    Navigate to quizzes and:
    - Answer multiple-choice questions
    - View explanations for correct answers
    - See your score
  </Step>
</Steps>

<Note>
  The AI processing time depends on the PDF size. A typical 20-page document takes 1-3 minutes to process.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="App crashes on startup">
    **Cause**: Missing or invalid Supabase credentials
    
    **Solution**:
    - Verify `lib/core/secrets.dart` exists and has correct values
    - Check that your Supabase project URL and anon key are correct
    - Ensure you ran `flutter pub get` to install dependencies
  </Accordion>
  
  <Accordion title="Worker not processing documents">
    **Cause**: Database connection issues or missing environment variables
    
    **Solution**:
    - Check `backend/.env` has all required variables:
      - `SUPABASE_URL`
      - `SUPABASE_KEY` (use service_role key, not anon key)
      - `GEMINI_API_KEY`
    - Verify the worker is running: `python worker.py`
    - Check worker logs for error messages
  </Accordion>
  
  <Accordion title="PDF upload fails">
    **Cause**: Storage bucket not configured or permission issues
    
    **Solution**:
    - Create a storage bucket named `pdfs` in Supabase dashboard
    - Make the bucket public or set appropriate policies
    - Check Row Level Security policies on the documents table
  </Accordion>
  
  <Accordion title="AI generates poor quality content">
    **Cause**: PDF text extraction issues or unclear source material
    
    **Solution**:
    - Ensure PDFs are text-based, not scanned images
    - Try PDFs with clear section headings and structured content
    - Adjust the AI prompt in `backend/worker.py` line 49-86 for better results
  </Accordion>
</AccordionGroup>
